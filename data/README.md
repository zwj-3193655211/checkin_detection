# 数据集目录结构

```
data/
├── raw/                    # 原始图片目录（统一存储位置）
├── labeled/                # 已标注图片
├── cache/
│   └── features/           # CLIP 特征缓存
├── labels.json             # 标签元数据
├── dataset_stats.json      # 数据集统计信息
└── split_config.json       # 数据集划分配置
```

## 图片命名格式
- 格式: `{学号}-{日期}.jpeg`
- 示例: `1-2026-04-13.jpeg`, `100-2026-04-24.jpeg`
- 日期范围: 2026-04-13 至 2026-04-24

## 标注说明
- 晨读: 正常晨读打卡
- 晨跑: 正常晨跑打卡
- 异常: 非晨读晨跑内容
- 不确定: 无法判断

## 数据集统计

- 总图片数：2040 张
- 晨读：1503 张 (73.7%)
- 晨跑：505 张 (24.8%)
- 异常：29 张 (1.4%)

数据划分：
- 训练集：1383 张 (67.8%)
- 验证集：189 张 (9.3%)
- 测试集：468 张 (22.9%)

## 数据管理

所有图片统一存储在 `raw/` 目录中：
- 训练、预处理、分析都从这个目录读取
- 使用 `integrate_new_data.py` 整合新数据
- 使用 `complete_labels.py` 处理标签
